# 更新日志

本项目所有值得记录的变更都会写在这里。

格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)。

## [1.1.0-build.5] - 2026-07-15

### 修复
- **App 与小组件数据同步**：快照写入使用短时后台任务保护，保存食材后立即返回主屏幕也能完成 App Group 写入；刷新限定到 FridgeTracker 小组件，并增加 15 分钟兜底时间线更新。
- **长保质期食材不显示**：小组件不再隐藏 30 天以后的未来食材，坚果等长保质期品类可以正常显示。
- **全新安装数据库失败**：在 iOS 27 全新安装环境中，打开 SwiftData 数据库前显式创建 App Group 的 `Library/Application Support` 目录。
- **测试污染用户数据**：App-hosted 单元测试与 UI 测试统一使用内存数据库，禁止测试进程覆盖真实的小组件快照。

### 验证
- Xcode Beta 27.0 / iOS 27.0 Simulator 中完成小、中、大三种小组件的 App → App Group → WidgetKit 端到端验证。
- 276 项单元测试全部通过，无失败、无跳过、无运行时警告。

## [1.1.0-build.3] - 2026-07-14

### 新增
- **坚果分类**:食材分类新增「🥜 坚果」，位于「零食」与「烘焙」之间。添加/编辑食材、列表筛选、历史筛选、设置默认存放区，以及小组件的分类过滤均已同步支持；过期提醒沿用常温干货的默认提前量（临期前 1 天）。

## [1.0.1-build.2] - 2026-07-10

### 修复
- **通知被拒后无声失效**:系统通知权限为「已拒绝」时，过期提醒会静默调度失败且无任何提示。现在设置页会检测 `.denied` 状态并显示橙色横幅「系统通知已关闭，过期提醒不会触发」，附带「前往系统设置开启」按钮直达系统设置；从设置返回或页面出现时自动刷新状态。

### 优化（小组件）
- **空状态居中**:大号组件在「暂无快过期食材」时改为水平垂直居中，并加入 ✓ 图标（此前为左对齐、贴顶显示）。
- **中号到期上色**:中号组件的到期文案恢复按状态着色（临期/过期），大号保持次要灰以维持信息层级。
- **Dynamic Type 防溢出**:开启辅助功能大字号时，大号组件展示条目上限由 8 降为 4，并关闭缩放系数，避免文字截断。
- **VoiceOver 合并朗读**:小号整卡与列表行合并为单一无障碍元素，一次读出「名称，分类，到期，存放区」，不再逐字段跳读。
- **小号深链到食材**:小号组件点击后直接深链到当前展示的那一条食材详情（此前统一跳首页）。

### 文档
- 新增 `docs/shipswift-review.md`:对照 ShipSwift recipe 的模块级 code review 报告（映射表、P0/P1/P2 分级、优点清单与依赖策略说明）。
