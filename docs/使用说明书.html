🗂️ 1. 四个标签页是什么

App 底部有四个标签，分别对应食材的整个生命周期：现在有什么、用完要补什么、以前买过什么、以及各类设置。

🧊 食材

你当前的库存。顶部有搜索框、分类筛选、右上角排序菜单和 「+」添加按钮。

每行显示图标、名称、分类·数量，以及彩色的到期状态。

安全 3 天内到期 已过期

🛒 补货

只放「用完后下次还要买」的食材，不是当前库存。

点 「从历史生成」 会自动把最近 30 天内吃掉过 ≥ 2 次的食材加进来。

🕘 历史

你添加过的所有食材形成的快捷模板库。

可搜索、按分类筛选，点一下就能用旧设置快速再添加。

⚙️ 设置

过期提醒、历史建议管理、数据导入导出、关于。

➕ 2. 添加食材

点食材页右上角 「+」 进入添加界面。下面几种方式可以混用，越用越省事。

  1. 快捷标签：顶部「最近添加」有一排横向小标签，点一下自动填好名称、分类、图标、存储区域 —— 你只需确认新的保质期。
  2. 拍包装识别：点「拍包装识别」用相机拍食品包装，自动识别名称和保质期（「保质期至 / 有效期至」或「生产日期 + 保质期」都能算）。首次会请求相机权限，被拒绝时会提示你去系统设置开启，或改用相册。
  3. 选包装图：从相册选一张包装照片，做同样的识别。
  4. 同名复用：手动输入一个以前用过的名称时，会自动套用上次的分类 / 图标 / 数量 / 存储区域，并按历史保质期推算新的到期日（到期日仍可手动改）。
  5. 填写字段并保存（见下表）。

📝 各字段说明

  • 食材名称（必填）
  • 分类：点选 chip（蔬菜 / 水果 / 肉类 / 海鲜 / 乳制品 / 蛋类 / 饮料 / 调味品 / 零食 / 烘焙 / 速冻食品 / 其他）
  • 自定义 Emoji 图标：可留空，留空时用分类默认图标
  • 存储区域：冷藏 / 冷冻 / 常温
  • 保质期日期（必填）；购买日期可选
  • 数量（可选）：支持 2瓶 这种总量，也支持 3/5瓶 这种「5 份里还剩 3 份」的写法
  • 备注（可选）
智能合并：保存时如果已存在「同名 + 同分类 + 同存储区 + 同到期日 + 同单位」的食材，会自动把数量合并到那一条，而不是新建一条重复记录。

🍽️ 3. 处理一样食材：吃掉 / 扔掉 / 加入补货 / 删除

两种入口：在食材列表左滑某一行，或点进食材详情页用按钮。

✅ 吃掉

数量减 1，记一笔「吃掉」记录。

同一食材被吃掉满 2 次会自动加入补货清单；如果是最后 1 份，会从库存移除。

🗑️ 扔掉

数量减 1，记一笔「扔掉」记录。

扔掉最后 1 份时，会问你要不要把它加入补货清单。

🛒 加入补货

手动把它放进补货清单。

如果已经在清单里，会提示「已在补货清单中」。

❌ 删除

仅用于「录错了」的情况。

不会记录吃掉或扔掉结果，也不影响补货建议。

关键区别：「吃掉 / 扔掉」是正常消耗，会进入历史并影响补货建议；「删除」是纠错。录错了请用删除，别用扔掉，以免污染历史和补货建议。

🔔 4. 过期提醒（通知）

让 App 在食材快过期前提醒你。

  1. 在设置页打开「开启过期提醒」，首次会请求通知权限。
  2. 选择「默认提前提醒」：分类默认 / 当天 / 1 / 2 / 3 / 7 天。
  3. 改了开关或提前天数后，App 会自动按当前库存重新排定所有提醒。

📅 「分类默认」的提前天数

食材分类提前提醒
肉类、海鲜提前 2 天
速冻食品提前 7 天
其他所有食材提前 1 天
提醒固定在当天本地时间早上 9:00 推送。已过期或提醒时间已经过去的食材不会再创建新通知，避免半夜或反复打扰。

📲 5. 桌面小组件 Widget

不打开 App 也能在桌面看到哪些食材快过期。

  • 支持小 / 中 / 大三种尺寸，显示最近即将到期的食材。
  • 长按小组件 →「编辑小组件」，可以为这个小组件单独选择只看某一个分类（例如只看「乳制品」），标题会随分类变化。
  • 点小组件里的某一行，会直接打开 App 并跳到那样食材的详情页。
  • 天数实时计算，每天自动刷新；在 App 里增删改食材也会即时刷新小组件。

🎛️ 6. 历史建议管理

在 设置 → 建议 → 历史建议管理 里微调「同名复用」的行为。

  • 对每个历史食材名调整复用默认值：分类、图标、存储区域、默认保质期天数。
  • 隐藏某个建议：隐藏后不在「最近添加 / 历史」里出现，也不会在你输入同名时自动套用。
  • 随时可以「恢复历史默认值」，撤销你的自定义覆盖。

💾 7. 备份与恢复

在 设置 → 数据 里手动备份或迁移数据。

📤 导出数据

把所有食材导出为一个 JSON 备份文件。

📥 导入数据

从 JSON 文件批量导入食材。

目前是手动 JSON 备份；iCloud 自动多设备同步是后续计划。

💡 8. 小贴士 / FAQ

  • 到期颜色： 绿色 = 安全， 橙色 = 3 天内到期， 红色 = 已过期。
  • 想让补货建议更准：正常吃完就点「吃掉」，而不是「删除」。
  • 数量写法：2瓶 表示 2 份；3/5瓶 表示 5 份里还剩 3 份。
  • 录错了用「删除」，别用「扔掉」，以免污染历史和补货建议。
