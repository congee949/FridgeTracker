优化更新说明

🧊 FridgeTracker 本次更新做了什么

一次基于代码调研的优化:修了几个会影响体验的正确性问题,顺手做了去重、缓存和清理。下面只讲这次改了什么。

📅 2026-06-01 ✅ 模拟器构建通过 🗂️ 改动 13 个文件 ↩️ 可一键回滚

一句话总览

把「凌晨推通知、保质期天数越看越少、从小组件偶发打不开详情、相机被拒无提示」等问题修掉,并在不改变你使用习惯的前提下让添加界面更跟手、代码更好维护。

体验修复 你会直接注意到 性能/稳定 更顺更稳,无需操作 代码质量 对使用无影响

✅ 你会注意到的变化

这些直接改变了使用体验。

体验修复 过期提醒改到早上 9:00

之前
提醒在当天凌晨 0 点推送(因为保质期日期默认是当天 0 点)。
现在
固定在本地时间早上 9:00推送;提醒时间已过去的不再补推。

体验修复 改通知设置后立即生效

之前
关掉提醒或改提前天数后,已排队的旧通知不更新,还会按老规则触发。
现在
一改开关或提前天数,就按当前库存重新排定全部提醒。

体验修复 保质期估算不再越看越少

之前
没填购买日期的食材,历史模板/补货里的「约 N 天」会每天减 1,慢慢变得不准。
现在
添加时冻结一次原始保质期,之后不再随时间衰减。

体验修复 从小组件点食材能可靠打开详情

之前
App 冷启动时数据还没加载完,点小组件偶发打不开对应食材详情。
现在
等数据加载完成再跳转,deep link 稳定生效。

体验修复 相机权限被拒有明确提示

之前
「拍包装识别」若相机权限被拒,只是黑屏/闪退式关闭,不知道为什么。
现在
弹出提示,可一键去系统设置开启,或改用「选包装图」从相册识别。

体验修复 小组件不再堆积陈旧过期食材

之前
小组件可能显示「已过期 180 天」之类几个月前的食材,挤占有用信息。
现在
过期超过 14 天的不再进入小组件。

体验修复 设置页「显示」分区改成诚实说明

之前
写死「首页排序:按到期日」,看着像设置项,其实排序是在食材页菜单里控制的,有误导。
现在
改成一句说明文字,讲清排序在哪切、小组件标题随分类变化。

体验修复 列表支持无障碍朗读

食材行加了合并的 VoiceOver 标签(名称、分类、到期状态一起读),图标 emoji 不再被单独念出。

⚙️ 性能与稳定性(无需操作)

看不见,但更顺、更稳。

性能/稳定 添加界面更跟手

「最近添加」「同名复用」的历史建议不再每次按键都全量扫描历史,改为缓存,只在数据真正变化时重算。历史食材多时输入更流畅。

性能/稳定 降低崩溃风险

去掉了多处对日期、URL 的强制解包(!),异常输入时安全回退而不是崩溃。

性能/稳定 OCR 与小组件写入更省

  • 包装识别的正则预编译一次,不再每次识别重新编译。
  • 小组件数据写入去掉了一次冗余排序。

🧹 代码质量(对使用无影响)

只为后续好维护,行为不变。

代码质量 重复逻辑收敛到一处

  • 到期文案与颜色(绿/橙/红)原本在行视图、详情页、小组件各写一份 → 合并为 App 与 Widget 共享的两个函数。
  • 「自动加入补货」「补货去重插入」逻辑原本在列表页和详情页各一份 → 收敛到 ReplenishmentItem 的静态方法,阈值常量也只剩一处。
  • 补货查询从「全量取出再内存过滤」改为 SwiftData 谓词。

代码质量 大文件瘦身 + 清理废弃代码

  • 677 行的添加界面把 body 拆成 6 个分区属性,编译更快、更好读(类型未跨文件搬动,避免工程引用风险)。
  • 删除了早已弃用的折叠标签栏滚动监听残留代码。

🔧 验证与回滚

  • 构建验证:用 iPhone 17 模拟器编译 App + 小组件,BUILD SUCCEEDED。
  • 版本控制:本次先建了 git 仓库,三个提交 baseline → optimize → docs。
  • 一键回滚到优化前: git reset --hard 6b56fe8
  • 决策记录:关键取舍写在仓库根的 implementation-notes.md。

⚠️ 升级注意

为修复「保质期越看越少」,给数据模型新增了一个字段 originalShelfLifeDays。这属于 SwiftData 的轻量自动迁移:已装旧版、库存里已有的食材,该字段为空,会自动回退到旧算法,不会丢数据。首次在真机升级后,建议确认一次库存数据完好。
